    - scan command accepts --verbose flag for detailed output
    - scan command accepts --overwrite flag to allow overwriting existing files
    - scan command accepts --no-files flag to skip file writing
    - In human mode: shows spinner during scan, then score + dimension table + next steps + file summary
    - In JSON mode: outputs ScanResult as JSON to stdout with no colors
    - In JSON mode: spinner/progress goes to stderr only
    - Files are written to output directory with summary of paths printed
    - If output files exist and --overwrite not set, error with suggestion
    - Invalid URL exits with code 1 and actionable error message
    - Successful scan exits with code 0
  </behavior>
  <action>
    1. Create `packages/cli/src/commands/scan.ts`:
       - Export `scanCommand` as a Commander Command
       - Arguments: `<url>` (required)
       - Options:
         - `-f, --format <format>` (choices: "human", "json", default: "human")
         - `-o, --output <dir>` (default: "./aeorank-output")
         - `--max-pages <n>` (parse as int)
         - `-v, --verbose` (boolean)
         - `--overwrite` (boolean)
         - `--no-files` (boolean)
       - Action handler:
         a. Validate URL (try `new URL(url)`, catch → helpful error with suggestion)
         b. Build ScanConfig from options (merge --max-pages into DEFAULT_CONFIG)
         c. Create spinner (no-op if --format json)
         d. Call `scan(url, config)` from @aeorank/core
         e. If `--format json`: `console.log(JSON.stringify(result, null, 2))` and exit
         f. If `--format human`:
            - Stop spinner
            - Print renderScore(result)
            - Print renderDimensionTable(result.dimensions)
            - Print renderNextSteps(result.dimensions)
         g. If not `--no-files`:
            - Check if output dir exists with files — if so and no `--overwrite`, exit with error
            - Create output dir (mkdir -p)
            - Write each GeneratedFile to output dir
            - Print file summary (list of paths written)
         h. Exit 0 on success

       - Error handling:
         - Wrap entire action in try/catch
         - On catch: stop spinner, call handleError(err), print message + suggestion to stderr, exit 1

    2. Wire scan command into `packages/cli/src/index.ts`:
       - Import scanCommand from ./commands/scan.ts
