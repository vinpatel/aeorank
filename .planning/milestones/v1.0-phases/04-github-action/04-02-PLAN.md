Create `action/.github/workflows/test.yml` — a CI workflow that tests the action itself when pushed to the `aeorank/action` repo.

**Workflow structure:**

```yaml
name: Test Action
on:
  push:
    branches: [main]
  pull_request:

permissions:
  checks: write
  pull-requests: write
  contents: read

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run AEOrank Action
        uses: ./
        with:
          url: https://example.com
          fail-below: 0
```

Key points:
- `uses: ./` references the action from the repo root (since action.yml will be at root of `aeorank/action`)
- Uses `fail-below: 0` so the test never fails on score (example.com will score low)
- Declares all required permissions
- Runs on push to main and on PRs (will test both Check Run and PR comment paths)
  </action>

    <automated>test -f action/.github/workflows/test.yml && grep -q "uses: ./" action/.github/workflows/test.yml && grep -q "checks: write" action/.github/workflows/test.yml && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>Self-test workflow exists, references action via `uses: ./`, declares required permissions, and scans example.com with fail-below: 0</done>
