    1. Install Stripe packages:
       `pnpm --filter @aeorank/web add stripe @stripe/react-stripe-js @stripe/stripe-js`

    2. Create `apps/web/lib/stripe.ts`:
       - Export Stripe server singleton: `new Stripe(process.env.STRIPE_SECRET_KEY!)`
       - Export plan config constant: `{ free: { name, scansPerMonth: 3 }, pro: { name, priceId: process.env.STRIPE_PRO_PRICE_ID, scansPerMonth: 50 }, api: { name, priceId: process.env.STRIPE_API_PRICE_ID, scansPerMonth: 500 } }`

    3. Create `apps/web/app/api/webhooks/stripe/route.ts`:
       - This is a PUBLIC route (excluded from Clerk in proxy.ts)
       - CRITICAL: Use `await request.text()` for raw body (NOT request.json() — breaks Stripe signature verification)
       - Verify signature with stripe.webhooks.constructEvent(body, signature, STRIPE_WEBHOOK_SECRET)
       - Handle events:
         - `checkout.session.completed`: Extract customer ID, subscription ID, and metadata (userId). Upsert into subscriptions table with plan = metadata.plan, status = "active"
         - `customer.subscription.updated`: Update plan, status, current_period_end
         - `customer.subscription.deleted`: Update status to "canceled"
       - Use createServiceSupabaseClient() (no Clerk session in webhook context)
       - Return 200 for handled events, 200 for unhandled (Stripe retries on non-2xx)

    4. Create `apps/web/app/(dashboard)/upgrade/actions.ts`:
       - "use server" directive
       - `createCheckoutSession(priceId: string, plan: string)`:
         - Get userId from auth()
         - Look up or create Stripe customer (store stripe_customer_id in subscriptions table)
         - Create checkout session with ui_mode: "embedded", line_items, mode: "subscription"
         - Set metadata: { userId, plan } so webhook can identify the user
         - Set return_url to /dashboard?session_id={CHECKOUT_SESSION_ID}
         - Return { clientSecret: session.client_secret! }
  </action>
