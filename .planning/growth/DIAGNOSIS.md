# Growth diagnosis — 2026-04-19

AEOrank has a real product, fresh V2 marketing (shipped 2026-04-19), a clean
competitive angle (open-source / MIT / generates fixes / deterministic /
CI-native vs Profound, Scrunch, Otterly, Peec, AthenaHQ's closed
monitoring), and **4 GitHub stars.**

This document captures the honest diagnosis of why the project is invisible
despite the product being sharp. Not a blame doc — an operational bottleneck
audit.

## Structural bottlenecks

1. **Polish-before-distribution cycle.** In the single session ending
   2026-04-19, we completed: a full V2 marketing redesign, an Inspector
   pass on the redesign, a competitor-messaging sharpen, and a full
   dashboard design-system refactor. All of it for a product with 4 stars.
   At this stage, polish has near-zero marginal return; a single good HN
   thread would have 1000× the return.

2. **Solo founder, all layers.** Writing TypeScript, doing design, writing
   copy, running infra, making strategy, operating deploys — all flows
   through one brain. That sequentializes everything. One part-time
   collaborator (even just on marketing) would roughly double throughput.

3. **Zero feedback loop.** 0 GitHub issues, 0 Discussions, no Discord. The
   roadmap is vibes-driven because real users haven't complained yet.

## Distribution bottlenecks (biggest single category)

4. **Nothing has been announced.** No tweet thread, no Show HN, no Product
   Hunt, no awesome-list entries, no Reddit posts, no dev.to cross-posts,
   no GitHub Topics tagged. Every one of those is free effort that
   compounds.

5. **No attribution loop.** The 9 generated files don't include
   `# Generated by AEOrank` anywhere. Every site shipping the output is
   invisible free advertising. One-line code change; permanent
   compounding.

6. **The scoreboard is unpublished.** We have scan data for 100 funded
   startups — real numbers, category-defining content asset — sitting in
   a JSON file. This is a full blog post + HN front-page candidate.
   Competitors would kill for this data and publish within 48 hours.

## Model / signal bottlenecks

7. **Pricing ratio is restrictive for OSS growth.** Starter = 1 site + 3
   scans/month. OSS virality usually needs a more generous free tier so
   people can evaluate, share, and brag about their score. Consider
   trading near-zero current revenue for real traction.

8. **No proof points.** Zero testimonials, zero named users, zero CI
   badges, zero case studies. Competitors inflate these (Otterly's
   "25,000 marketing professionals"). We're correctly refusing to
   fabricate — but the credibility gap is real. One consented named
   early-adopter would close it.

9. **Category awareness is low.** "AEO" as a term is ~18 months old. Most
   of the addressable market doesn't know to search for this yet. Either
   dominate the SEO for the category now, before Profound outspends us
   (they raised ~$155M), or get buried when they do.

## Opportunity-cost bottlenecks

10. **80 Dependabot alerts** on main (3 critical, 24 high, 42 moderate,
    11 low). Any technical evaluator who looks seriously sees this as a
    "project isn't maintained" signal. One-afternoon fix.

11. **The 8 vs 9 files discrepancy.** Site copy claims 9 generated files;
    the build log says 8. A technical evaluator will catch this within 60
    seconds of installing. Credibility hole that costs nothing to close.

12. **No CI badge.** 288 tests pass. Without a shields.io badge on the
    README, nobody knows.

## The single-sentence version

The product is ready. The distribution hasn't started. That's the single
most fixable thing, and it's entirely in our control.
