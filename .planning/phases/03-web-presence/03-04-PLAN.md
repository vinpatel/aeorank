  <action>
  Create two GitHub Actions workflow files.

  **1. `.github/workflows/deploy-marketing.yml`:**
  ```yaml
  name: Deploy Marketing Site

  on:
    push:
      branches: [main]
      paths:
        - 'apps/marketing/**'
        - '.github/workflows/deploy-marketing.yml'
    workflow_dispatch:

  permissions:
    contents: read
    pages: write
    id-token: write

  concurrency:
    group: pages-marketing
    cancel-in-progress: false

  jobs:
    build:
      runs-on: ubuntu-latest
      steps:
        - name: Checkout
          uses: actions/checkout@v5

        - name: Build with Astro
          uses: withastro/action@v5
          with:
            path: apps/marketing
            package-manager: pnpm@10.32.1

    deploy:
      needs: build
      runs-on: ubuntu-latest
      environment:
        name: github-pages
        url: ${{ steps.deployment.outputs.page_url }}
      steps:
        - name: Deploy to GitHub Pages
          id: deployment
          uses: actions/deploy-pages@v4
  ```

  **2. `.github/workflows/deploy-docs.yml`:**
  ```yaml
  name: Deploy Docs Site

  on:
    push:
      branches: [main]
      paths:
        - 'apps/docs/**'
        - '.github/workflows/deploy-docs.yml'
    workflow_dispatch:

  concurrency:
    group: pages-docs
    cancel-in-progress: false

  jobs:
    build-and-deploy:
      runs-on: ubuntu-latest
      steps:
        - name: Checkout
          uses: actions/checkout@v5

        - name: Setup pnpm
          uses: pnpm/action-setup@v4
          with:
            version: 10.32.1

        - name: Setup Node.js
          uses: actions/setup-node@v4
          with:
            node-version: 22
            cache: pnpm

        - name: Install dependencies
          run: pnpm install --frozen-lockfile

        - name: Build docs site
          run: pnpm --filter @aeorank/docs build

        - name: Deploy to docs repo
          uses: peaceiris/actions-gh-pages@v4
          with:
            deploy_key: ${{ secrets.DOCS_DEPLOY_KEY }}
            external_repository: aeorank/docs
            publish_dir: apps/docs/dist
            cname: docs.aeorank.dev
  ```

  **Key design decisions:**
  - Marketing uses `withastro/action@v5` (official, handles pnpm detection, caching)
  - Docs uses manual pnpm/node setup because it needs to push to an external repo
  - Path-based triggers prevent unnecessary builds (marketing changes don't trigger docs deploy)
  - `workflow_dispatch` allows manual triggers for both
  - Concurrency groups prevent simultaneous deploys
  - Marketing uses GitHub Pages deployment environment (official API)
  - Docs uses `peaceiris/actions-gh-pages` to push to a separate repo's `gh-pages` branch

  Create the `.github/workflows/` directory if it doesn't exist.

  <name>Task 2: User configures GitHub Pages and DNS</name>

  <action>
  Present the user with the setup checklist. These steps require human action in GitHub and DNS settings.

  **GitHub Setup (main repo):**
  1. Go to repo Settings → Pages → Source: select "GitHub Actions"
  2. Set custom domain to `aeorank.dev`

  **GitHub Setup (docs repo):**
  1. Create `aeorank/docs` repository on GitHub (can be empty, public)
  2. Generate SSH deploy key: `ssh-keygen -t ed25519 -C "docs-deploy" -f docs_deploy_key -N ""`
  3. Add `docs_deploy_key.pub` (public key) as a Deploy Key with write access in `aeorank/docs` repo settings
  4. Add `docs_deploy_key` (private key) as a secret named `DOCS_DEPLOY_KEY` in the main `aeorank/aeorank` repo settings → Secrets → Actions
  5. Enable GitHub Pages in `aeorank/docs` repo: Settings → Pages → Source: "Deploy from a branch" → Branch: `gh-pages`
  6. Set custom domain to `docs.aeorank.dev`

  **DNS Setup (at domain registrar):**
  1. Add A records for `aeorank.dev` pointing to GitHub Pages IPs:
     - 185.199.108.153
     - 185.199.109.153
     - 185.199.110.153
     - 185.199.111.153
  2. Add CNAME record: `docs.aeorank.dev` → `aeorank.github.io`

  **Note:** DNS propagation may take up to 24 hours. Sites will work at the github.io URLs immediately.
