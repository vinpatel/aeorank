  <action>
    1. Create `apps/web/lib/qstash.ts`:
       - Export QStash Client singleton: `new Client({ token: process.env.QSTASH_TOKEN! })`

    2. Create `apps/web/app/api/scan/route.ts` (POST handler):
       - Import auth from @clerk/nextjs/server — reject if no userId (401)
       - Parse body: `{ url: string }`
       - Call validateScanUrl(url) — return 400 with error message on failure
       - Create or find site in Supabase `sites` table (upsert by user_id + url)
       - Insert pending scan record in `scans` table (user_id, site_id, status: "pending")
       - Enqueue QStash job: `qstash.publishJSON({ url: NEXT_PUBLIC_APP_URL + "/api/scan/process", body: { scanId, url } })`
       - Return 202 with `{ scanId, siteId }`

    3. Create `apps/web/app/api/scan/process/route.ts` (QStash callback):
       - This is a PUBLIC route (excluded from Clerk in proxy.ts)
       - Verify QStash signature using @upstash/qstash Receiver (QSTASH_CURRENT_SIGNING_KEY + QSTASH_NEXT_SIGNING_KEY)
       - Return 401 if signature invalid
       - Parse body: `{ scanId, url }`
       - Use createServiceSupabaseClient() (service role key, no Clerk token — QStash has no user session)
       - Update scan status to "running"
       - Call `scan(url)` from @aeorank/core
       - On success: Update scan record with score, grade, dimensions (as JSONB), files (as JSONB array of {name, content}), pages_scanned, duration_ms, status "complete"
       - On error: Update scan record with status "error" and error message
       - Return 200

    IMPORTANT: The process route uses service role Supabase client because QStash callbacks have no Clerk session. RLS is bypassed by service role — the scan's user_id was set at enqueue time.
