  <action>
Create `action/action.yml` as a composite GitHub Action. This file will be copied to the root of the standalone `aeorank/action` repo for Marketplace publishing.

**Metadata:**
- name: `"AEOrank — AEO Scanner"`
- description: `"Scan any URL for AI Engine Optimization (AEO) score. Posts results as a GitHub Check and PR comment."`
- author: `"AEOrank"`
- branding: icon `"bar-chart-2"`, color `"blue"`

**Inputs:**
- `url` (required): URL to scan
- `token` (required: false, default: `${{ github.token }}`): GitHub token for API calls
- `fail-below` (required: false, default: `'0'`): Fail Check if AEO score drops below this value (0 = never fail)

**Steps (using: "composite"):**

**Step 1 — Run AEOrank scan** (id: `scan`, shell: bash):
```bash
RESULT=$(npx aeorank@latest scan "${{ inputs.url }}" --format json --no-files 2>/dev/null)
echo "result<<AEORANK_EOF" >> $GITHUB_OUTPUT
echo "$RESULT" >> $GITHUB_OUTPUT
echo "AEORANK_EOF" >> $GITHUB_OUTPUT
# Extract score for quick access
SCORE=$(echo "$RESULT" | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8');console.log(JSON.parse(d).score)")
echo "score=$SCORE" >> $GITHUB_OUTPUT
GRADE=$(echo "$RESULT" | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8');console.log(JSON.parse(d).grade)")
echo "grade=$GRADE" >> $GITHUB_OUTPUT
```

**Step 2 — Post Check Run** (uses: `actions/github-script@v8`):
- `github-token: ${{ inputs.token }}`
- Script logic (in the `script:` field):
  1. Parse the scan result JSON from `${{ steps.scan.outputs.result }}`. IMPORTANT: Do NOT use template literal interpolation for the JSON — instead, use an environment variable. Set `env: SCAN_RESULT: ${{ steps.scan.outputs.result }}` and parse with `JSON.parse(process.env.SCAN_RESULT)`.
  2. Build a markdown dimension table from `result.dimensions` using status emoji mapping: `{ pass: '✅', warn: '⚠️', fail: '❌' }`. Table columns: Dimension | Score | Status | Recommendation.
  3. Determine the correct SHA: `const sha = context.eventName === 'pull_request' ? context.payload.pull_request.head.sha : context.sha;`
  4. Compute conclusion: if `failBelow > 0 && score < failBelow` → `'failure'`; else if `score >= 70` → `'success'`; else if `score >= 40` → `'neutral'`; else → `'failure'`.
  5. Call `github.rest.checks.create()` with: owner, repo, head_sha, name `'AEOrank Score'`, status `'completed'`, conclusion, output with title `AEO Score: ${score}/100 (${grade})`, summary with score/grade/url, and text with the dimension table markdown.
  6. Export `core.exportVariable('AEORANK_TABLE', table)` so the PR comment step can use it.

**Step 3 — Find existing PR comment** (if: `github.event_name == 'pull_request'`):
- id: `fc`
- uses: `peter-evans/find-comment@v3`
- with: token, issue-number `${{ github.event.pull_request.number }}`, comment-author `'github-actions[bot]'`, body-includes `'<!-- aeorank-score -->'`

**Step 4 — Upsert PR comment** (if: `github.event_name == 'pull_request'`):
- uses: `peter-evans/create-or-update-comment@v5`
- with: token, comment-id `${{ steps.fc.outputs.comment-id }}`, issue-number `${{ github.event.pull_request.number }}`, edit-mode `replace`
- body: A multiline string starting with `<!-- aeorank-score -->` hidden marker, then `## AEOrank Score: ${{ steps.scan.outputs.score }}/100 (${{ steps.scan.outputs.grade }})`, followed by the dimension table (reference via `${{ env.AEORANK_TABLE }}`), and a footer line: `*Scanned ${{ inputs.url }} · [What is AEOrank?](https://aeorank.dev)*`

**Key pitfalls to avoid (from research):**
- Use `context.payload.pull_request.head.sha` on PR events, NOT `context.sha` (merge commit SHA)
- Use `env:` block to pass large JSON to github-script, NOT template interpolation
- Use `AEORANK_EOF` as delimiter (not generic `EOF`) to avoid collision
- The hidden HTML comment `<!-- aeorank-score -->` must be the FIRST thing in the PR comment body for find-comment to work
